Approximate Duplicate Detection

Description

The Approximate Duplicate Detection step allows the user to compute groups of records that represent the same real-world entity.
These groups are built based on rules provided by the user to define how fields are compared to compute similarity between records.

The step outputs two columns: The first column corresponds to the grouping of records. Each group contains the records that correspond to one real-world entity. The group identifier is the index of the first record of the group. The second column corresponds to the average similarity between records of the group.

Options

The following table describes the options associated with the Approximate Duplicate Detection step

Option Description
Step Name The name of this step as it appears in the transformation workspace. This name must be unique within the transformation.
Partitioning Attributes Table The field or fields that will be used to partition the records with a blocking approach. If no attribute is chosen, a cartesian product will be made. The attributes chosen for blocking should have a high confidence of being correct throughout the dataset.
Similarity Threshold A value between 0 and 1 that corresponds to the minimum value of similarity that must exist between two records for them to be considered approximate duplicates, where 0 corresponds to the records being completely different and 1 corresponds to the records being exactly alike.
Fields Table In this table, the user selects which attributes from the dataset are to be used in the matching process, as well as the similarity measure to use and the weight that each attribute will have in the overall similarity value.
Grouping Column Name The name of the output column that contains the group to which each the record belongs to.
Similarity Column Name The name of the output column that contains the average similarity of the record in the row with the remaining elements of its group.
Remove Duplicates By selecting this option, only the first record of each group is present in the step output.
Remove Non-Duplicates By selecting this option, the records that do not match any other records are removed from the step output.
